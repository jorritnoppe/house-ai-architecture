# House AI Architecture

Local-first house AI architecture focused on safe automation, voice interaction, telemetry, and modular service orchestration.

## Project focus
This project explores how a local AI system can:
- reason over house telemetry
- route requests through safe APIs
- control automation through explicit guarded actions
- support voice interaction across rooms
- grow toward a context-aware house assistant

## Architecture themes
- local LLM inference
- Flask-based control layer
- route/service/tool separation
- safety and approval boundaries
- telemetry-driven reasoning
- multi-room voice pipeline groundwork

## Public repo note
This repository is a sanitized architecture and implementation showcase.
Environment-specific deployment values, private runtime state, and local secrets are intentionally excluded.

## Main subsystems
- route layer
- service layer
- tool layer
- telemetry integration
- voice pipeline
- approval and safety model

## Long-term direction
A local AI assistant that can understand state, history, context, and safe automation boundaries.
